# 🧩 Audits and Assessments

## 🔍 1. Overview

**Audit:**  
Organizasyonun bilgi sistemleri, uygulamaları ve güvenlik kontrollerinin **sistematik olarak değerlendirilmesi**.

**Assessment:**  
Zafiyetleri ve riskleri belirlemek için yapılan **ayrıntılı analiz**.

🧠 **Amaç:**  
- Güvenlik önlemlerini doğrulamak  
- Uyum (Compliance) sağlamak  
- Güvenlik açıklarını tespit etmek  
- Sürekli iyileştirme sağlamak  

---

## 🏛️ 2. Types of Audits

| Type | Description | Example |
|------|--------------|----------|
| **Internal Audit** | Kurum içi ekip tarafından yapılır. | Data protection policy incelemesi |
| **External Audit** | Üçüncü taraf (bağımsız) denetçiler tarafından yapılır. | PCI DSS uyumluluğu denetimi |

### 🎯 Purposes of Audits
- **Identify Gaps:** Politikalar, prosedürler, kontroller  
- **Ensure Compliance:** GDPR, HIPAA, PCI DSS  
- **Improve Security Posture**  

---

## ⚙️ 3. Types of Assessments

| Assessment Type | Purpose |
|------------------|----------|
| **Risk Assessment** | Olası tehditlerin ve etkilerinin değerlendirilmesi |
| **Vulnerability Assessment** | Sistem zafiyetlerinin tespiti |
| **Threat Assessment** | Organizasyonu etkileyebilecek tehditlerin analizi |

💡 Genellikle yeni sistem kurulmadan veya büyük değişikliklerden önce yapılır.  

---

## 🧭 4. Internal Audits & Assessments

**Internal Audit:**  
Kurumun kendi ekibi tarafından yürütülür.  
Değerlendirilen alanlar:
- Network security  
- Access controls  
- Incident response  
- Data protection  

**Process:**  
1. Policy ve prosedürleri gözden geçir  
2. Erişim haklarını incele  
3. Kontrollerin etkinliğini test et  
4. Bulguları dokümante et ve öneri ekle  

**Audit Committee:**  
- Yönetim kurulu üyelerinden oluşur  
- İç/dış denetimleri ve uyumu izler  
- Finansal raporlama ve kontrolleri değerlendirir  

**Internal Assessment:**  
Kurum içi güvenlik duruşunun proaktif değerlendirmesi.  
- Threat Modeling  
- Vulnerability Scanning  
- Risk Evaluation  
- Mitigation Strategies  

🧩 *MCIT Cybersecurity Self-Assessment* örneği:  
Yes/No sorular, yorum alanı ve aksiyon sahipleri içerir.  

---

## 🌐 5. External Audits & Assessments

**External Audit:**  
Bağımsız kurumlar tarafından yürütülür, objektiflik sağlar.  
Odağı:
- Data protection  
- Network security  
- Incident response  

**External Assessment:**  
Dış kaynaklı analiz; araçlar ve manuel testlerle yapılır.  
Kapsam:
- Risk / Vulnerability / Threat Assessments  
- HIPAA, PCI DSS, GDPR uyumu  

**Regulatory Compliance:**  
Uyulması gereken standartlar:  
- **NIST CSF**  
- **HIPAA**  
- **GDPR**  
- **PCI DSS**

**Independent Third-Party Audits:**  
- Güvenilirlik ve şeffaflık sağlar  
- Müşteriler, paydaşlar ve regülatörlerle güven oluşturur  

---

## 🧨 6. Penetration Testing (Pentesting)

**Definition:**  
Gerçek siber saldırı simülasyonu; sistemdeki zafiyetlerin tespiti.

| Type | Description |
|------|--------------|
| **Physical** | Fiziksel güvenlik testleri (kilit, kamera, erişim kartı) |
| **Offensive (Red Team)** | Saldırı simülasyonu, zafiyet tespiti |
| **Defensive (Blue Team)** | Savunma, izleme, olay müdahalesi |
| **Integrated (Purple Team)** | Red + Blue iş birliği, eğitim ve geliştirme odaklı |

**Tools:** Metasploit, Nmap, Burp Suite  

---

## 🕵️ 7. Reconnaissance in Pentesting

**Reconnaissance:**  
Saldırı öncesi bilgi toplama aşaması.

| Type | Description |
|------|--------------|
| **Passive** | Dolaylı bilgi toplama (OSINT, WHOIS) |
| **Active** | Hedefle doğrudan etkileşim (Nmap port taraması) |

### Environment Types
- **Known Environment:** İçeriden bilgi mevcut  
- **Partially Known:** Sınırlı bilgi  
- **Unknown Environment:** Dış saldırgan senaryosu  

---

## 🧾 8. Attestation of Findings

**Definition:**  
Denetim veya assessment sonucunun doğruluğunu **resmen belgeleyen yazılı onay**.

| Type | Purpose |
|------|----------|
| **Software Attestation** | Yazılımın bütünlüğünü doğrular |
| **Hardware Attestation** | Donanımın değişmediğini garanti eder |
| **System Attestation** | Sistem güvenlik duruşunu doğrular |

**Audit Context:**  
- **Internal:** Politika ve kontrollerin etkinliğini kanıtlar  
- **External:** Finansal / regülasyon uyumunun doğrulanması  
- **Outcome:** Güven, şeffaflık ve hesap verebilirlik sağlar  

---

📘 **Quick Summary**
- **Audits** → Sistematik inceleme  
- **Assessments** → Risk ve zafiyet analizi  
- **PenTesting** → Gerçek saldırı simülasyonu  
- **Reconnaissance** → Bilgi toplama aşaması  
- **Attestation** → Bulguların doğrulanması
